Conceptos básicos

¿Qué son Git y GitHub?

No son lo mismo — y entender la diferencia lo cambia todo.

🗂️

Git

Imagina que estás escribiendo una tesis. Git es como tener un botón mágico de "guardar versión" que recuerda cada cambio que hiciste: qué modificaste, cuándo y por qué.

Si algo sale mal, puedes regresar a cualquier punto anterior. ¡Vive en tu propia computadora y no necesita internet!

💡 Analogía: Es el "Ctrl+Z" más poderoso del mundo, pero para tu código.
☁️

GitHub

GitHub es una plataforma en internet donde guardas tu código (con todo el historial de Git) en la nube. Es como Google Drive, pero diseñado para programadores.

Además, te permite trabajar con otras personas en el mismo proyecto sin pisarse los pies. ¡Como Google Docs para código!

💡 Analogía: Es la bodega en la nube donde guardas y compartes tu trabajo.
Comparación

Git vs GitHub

¿Cuál es la diferencia real?

🗂️ Git ☁️ GitHub
¿Qué es? Programa que instalas en tu PC Sitio web / plataforma en línea
¿Dónde vive? En tu computadora local En internet (la nube)
¿Para qué sirve? Controlar versiones de tu código Compartir y colaborar en proyectos
¿Necesita internet? ❌ No ✅ Sí
¿Trabaja solo? ✅ Funciona sin GitHub ❌ Necesita Git para funcionar

👉 En resumen: Git es la herramienta — GitHub es donde la usas con otros.

Proceso

¿Cómo funciona en la vida real?

El flujo de trabajo típico paso a paso.

1
Creas o clonas un repositorio

Un repositorio ("repo") es la carpeta del proyecto. Lo inicias desde cero con git init o lo copias de GitHub con git clone.

↓
2
Creas una rama (branch)

Una rama es una copia paralela donde puedes trabajar sin afectar el código principal de los demás. Como un borrador propio.

↓
3
Haces cambios y los confirmas (commit)

Modificas archivos y los "confirmas" con un mensaje. Es como guardar con una nota que explica qué cambiaste y por qué.

↓
4
Subes a GitHub (push)

Envías tus commits al repositorio en internet para que otros los vean y descarguen.

↓
5
Creas un Pull Request

Le dices a tu equipo: "Revisad lo que hice y, si está bien, incluidlo en el proyecto principal." Es la base del trabajo colaborativo.
